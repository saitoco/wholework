## 主要ファイル（Required）

### Skills

Skills は現在プライベートリポジトリから積極的にマイグレーション中。各 Skill は `skills/<skill-name>/SKILL.md` に配置。多くの Skill はサブフェーズや専門ガイドライン用の補助 `.md` ファイルも含む（例: `external-review-phase.md`、`codebase-search.md`）。現在のリストは `skills/` ディレクトリを参照。

### Modules

主要モジュール:
- `modules/verify-patterns.md` — verify command パターン精度ガイドライン
- `modules/verify-classifier.md` — マージ後条件の検証可能性分類
- `modules/verify-executor.md` — verify command の変換と実行
- `modules/worktree-lifecycle.md` — 全 Skill 共有の worktree Entry/Exit ライフサイクル
- `modules/test-runner.md` — 品質チェック実行と結果分析
- `modules/size-workflow-table.md` — Size→ワークフロー判定テーブル
- `modules/detect-config-markers.md` — `.wholework.yml` 設定検出
- `modules/adapter-resolver.md` — 3 層アダプター解決（プロジェクトローカル → ユーザーグローバル → バンドル）
- `modules/opportunistic-verify.md` — Skill 完了時のオポチュニスティック検証
- `modules/doc-checker.md` — ドキュメント整合性チェッカー
- `modules/skill-help.md` — Skill 共有 `--help` 出力フォーマッター
- `modules/skill-dev-checks.md` — クロス Skill 整合性バリデーション
- `modules/codebase-analysis.md` — `/doc` deep モード用コードベース分析
- `modules/title-normalizer.md` — Issue タイトル正規化
- `modules/ambiguity-detector.md` — Issue 記述の曖昧性検出
- `modules/review-output-format.md` — レビュー出力フォーマット
- `modules/review-type-weighting.md` — レビュータイプ重み付け設定
- `modules/project-field-update.md` — GitHub Projects フィールド更新
- `modules/browser-adapter.md` — ブラウザベース検証アダプター
- `modules/browser-verify-security.md` — ブラウザ検証セキュリティチェック
- `modules/lighthouse-adapter.md` — Lighthouse パフォーマンス監査アダプター
- `modules/measurement-scope.md` — 測定スコープ定義
- `modules/next-action-guide.md` — 全 Skill 統一の次アクションガイダンス

### インストール

Wholework は 2 つのインストール方法をサポート:

**Marketplace インストール**（プライマリ）:

```sh
/plugin marketplace add saitoco/wholework
/plugin install wholework@saitoco-wholework
```

Marketplace マニフェストは `.claude-plugin/marketplace.json`（name: `saitoco-wholework`）にあります。

**開発インストール**（ローカル）:

```sh
git clone https://github.com/saitoco/wholework.git
cd wholework
./install.sh
claude --plugin-dir <path-to-wholework>
```

Skill は `wholework:<skill-name>` として検出されます。Claude Code は実行時に `${CLAUDE_PLUGIN_ROOT}` をプラグインディレクトリに設定し、Skill とモジュールはこれを使用してスクリプトやモジュールを参照します。

**なぜ `./install.sh` が必要か？** `.claude/settings.json` は `.claude/settings.json.template` から、ユーザーの実際の `$HOME` を `${HOME}` に代入して生成されます。Claude Code は `permissions.allow` 内の `${HOME}` や `~/` を展開しないため、各開発者がローカルでテンプレートを実体化する必要があります。生成された `.claude/settings.json` は gitignore されます。clone 後に一度 `./install.sh` を実行し、`.claude/settings.json.template` が変更された場合は `git pull` 後に再度実行してください。
