# 翻訳ワークフロー

このドキュメントは、英語のソースドキュメントに対して `docs/ja/` ミラーファイルを同期するルールを定義します。トップレベルの `docs/*.md` ファイルを変更するスキルは、このドキュメントを参照し、以下に記述する同期手順を適用する必要があります。

## 同期タイミング

スキル実行中（`/code`、`/spec`、`/review` など）にトップレベルの `docs/*.md` ファイルが追加または変更された場合は、`docs/ja/` ミラーファイルを同期してください。

## 除外対象

以下のパスは同期義務から**除外**されます：

- `docs/spec/` — Issue 仕様書（使い捨て、翻訳不要）
- `docs/reports/` — 監査・最適化レポート（翻訳不要）
- `docs/ja/` — 翻訳ファイル自体（再帰的な同期は行わない）

## 同期手順

1. 変更ファイルリストの中から、トップレベルの `docs/*.md` ファイルを特定する（上記の除外パスを除く）。
2. 特定した各ファイルに対して、対応する `docs/ja/<filename>.md` ミラーの場所を確認する。
3. ミラーが存在しない場合は新規作成する。
4. 英語の変更内容を反映するようにミラーを更新し、日本語で記述する。英語の元ファイルと一致する構造・見出し・書式を維持する。

## このドキュメントを参照するスキル

| スキル | SKILL.md パス | トリガータイミング |
|--------|--------------|------------------|
| `/code` | `skills/code/SKILL.md` | ドキュメント整合性チェック後（Step 9） |
| `/spec` | `skills/spec/SKILL.md` | 変更ファイルリスト構築時（Step 10） |
| `/review` | `skills/review/SKILL.md` | ドキュメント整合性レビュー中（レビューチェックリスト） |
