# Issue #72: feat: /doc translate に言語指定引数を追加

## Overview

`/doc translate` に言語コード引数 `{lang}` を追加し、任意の言語でドキュメントを翻訳生成できるようにする。現在は日本語（`ja`）がハードコードされているが、BCP 47 / ISO 639-1 形式の言語コードを引数として受け取ることで汎用化する。引数なし実行時は Usage メッセージを表示して中止する。

## Changed Files

- `skills/doc/SKILL.md`: 5箇所の変更
  1. frontmatter `description`: "Japanese translations" → 汎用表現に変更
  2. Command Routing: `translate` → `translate {lang}` 形式のパース追加、引数なし時 Usage 表示
  3. translate セクション（見出し・本文・翻訳指示・出力パス・コミットメッセージ）: 日本語ハードコードを `{lang}` テンプレートに変更、BCP 47 / ISO 639-1 言及追加
  4. sync --deep 除外リスト: `README.ja.md` / `docs/ja/` を言語コードパターンに汎用化
  5. (関連) Command Routing の Usage 表示文: `translate` → `translate {lang}` に更新
- `docs/structure.md`: `ja/` ディレクトリと `README.ja.md` の説明を多言語対応に更新
- `docs/workflow.md`: `/doc translate` の説明文を言語指定引数対応に更新

## Implementation Steps

1. `skills/doc/SKILL.md` frontmatter description を更新: "`/doc translate` for generating Japanese translations" → "`/doc translate {lang}` for generating multi-language translations" (→ 受入基準 10)

2. Command Routing を更新: (→ 受入基準 1, 2, 8)
   - `translate {lang}` 形式のルーティングを追加（`translate` で始まる場合: `{lang}` を抽出してセクションへ）
   - 引数なし(`translate` のみ)の場合: `Usage: /doc translate {lang}` を表示して終了
   - Usage 表示文の `translate` を `translate {lang}` に更新

3. translate セクションを汎用化: (→ 受入基準 3, 4, 5, 6, 7, 11)
   - セクション見出し: `## translate — Japanese Translation Generation` → `## translate — Translation Generation`
   - セクション導入文: `Execute when ARGUMENTS is translate {lang}. ...` に変更、`{lang}` 変数について BCP 47 / ISO 639-1 準拠・LLM 解釈委ねの旨を記載
   - Step 1（Identify Target Files）前に Step 0 追加: `{lang}` が空の場合 Usage 表示して終了
   - 出力パスルール: `docs/ja/` → `docs/{lang}/`、`README.ja.md` → `README.{lang}.md` に変更
   - 翻訳指示: "Translate the entire file content to Japanese" → "Translate the entire file content to the target language (language code: `{lang}`)" に変更
   - Relative Link Rewriting Rules: `docs/ja/{name}.md` → `docs/{lang}/{name}.md`、`README.ja.md` → `README.{lang}.md` に変更
   - コミットメッセージ: `docs: regenerate Japanese translations` → `docs: regenerate {lang} translations` に変更
   - `git add` コマンド: `README.ja.md docs/ja/` → `README.{lang}.md docs/{lang}/` に変更
   - Step 2 (Prepare Output Directory): `mkdir -p docs/ja` → `mkdir -p docs/{lang}` に変更
   - Completion Report メッセージ: "Japanese translation generation complete." → 言語コードを含む汎用表現に変更

4. sync --deep 除外リストを汎用化（line 314）: (→ 受入基準 9)
   - `README.ja.md at project root, files under docs/ja/` → `README.{lang}.md at project root (language code directory pattern), files under docs/{lang}/` もしくは "language-code-named directories under docs/" のようなパターン記述に変更

5. `docs/structure.md` と `docs/workflow.md` を更新: (→ doc-checker 判定)
   - `docs/structure.md` line 44: `ja/  # Japanese translations (docs/ja/) generated by /doc translate` → `{lang}/  # Language translations (docs/{lang}/) generated by /doc translate {lang}` に更新
   - `docs/structure.md` line 48: `README.ja.md  # Japanese translation of README.md` → `README.{lang}.md  # Language translation of README.md` に更新
   - `docs/workflow.md` line 28: `/doc translate` の説明を言語引数対応の表現に更新
