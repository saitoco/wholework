# review-output-format module

## Purpose

Centrally manages the output format specification (shared finding format patterns + path/line specification rules) shared by the three agents: review-bug, review-light, and review-spec.

## Input

Finding content generated by the calling agent.

## Processing Steps

Structure output following the shared finding format patterns and path/line specification rules below.

### Shared Finding Format Pattern

```
**[Category] filename:near-line-number**
- path: file path (relative to repository root; null if not identifiable)
- line: line number (relevant line in diff; null if not identifiable)
Finding content. Severity: MUST / SHOULD / CONSIDER

Recommended action:
(specific fix suggestion)
```

### path/line Specification Rules

- `path` is the relative path from the repository root (e.g., `scripts/run-code.sh`)
- `line` is the line number in the post-change file (identified from the PR diff)
- For findings where path/line cannot be identified (design-level, architecture-level), output `path: null`, `line: null`
- If path can be identified but line cannot, use `line: null`
- Values should be output as raw values without backtick wrapping (e.g., `path: scripts/run-code.sh`, `line: 42`, `path: null`)

## Output

The calling agent outputs findings with the above format applied.
