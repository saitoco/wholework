# label-conventions

SSoT for Wholework label namespace conventions and bare-namespace exceptions.

## Purpose

Define the exhaustive namespace rules for Wholework-managed GitHub labels so that:
- New label additions follow the established namespace structure.
- Bare-namespace exceptions remain minimal and explicitly documented.
- Skills and scripts can validate label names against this SSoT.

## Label Namespace Table

| Namespace | Purpose | Exclusivity |
|-----------|---------|-------------|
| `phase/*` | Workflow phases (issue / spec / ready / code / merge / review / verify / done / skill) | Reserved; no other use |
| `type/*` | Issue type (bug / feature / task) | Reserved; no other use |
| `size/*` | Issue complexity (XS – XL) | Reserved; no other use |
| `audit/*` | Origin tracking for Issues auto-generated by `/audit` subcommands | Reserved; no other use |
| `retro/*` | Retrospective-derived Issue origin tracking | Reserved; no other use |
| (bare) `triaged` | Indicates `/triage` has been executed on the Issue | **Sole bare-namespace exception** — see below |

## Bare Namespace Exception: `triaged`

`triaged` is the only label in Wholework that does not follow the `namespace/name` convention.

**Why bare namespace is retained:**

1. The label name directly corresponds to the `/triage` skill name — the relationship is self-evident.
2. Changing to `state/triaged` would require migrating all existing Issues and updating every skill/script that references the label name.
3. The user community has established visual recognition of `triaged` as a standalone label.

**Rule:** Adding new bare-namespace labels is prohibited. `triaged` is the explicit and permanent exception.

Any future label additions must use the `namespace/name` form. If no existing namespace fits, create a new namespace first (update this file and `scripts/setup-labels.sh`).

## Relation to setup-labels.sh

`scripts/setup-labels.sh` is the SSoT for label *definitions* (name, color, description, creation condition).
This file (`modules/label-conventions.md`) is the SSoT for label *naming rules* (namespace structure, bare-namespace policy).

Both must be kept in sync: a label defined in `setup-labels.sh` must belong to a namespace listed here (or be `triaged`).
